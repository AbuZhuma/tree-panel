# Деплой Tree Builder Panel

Домен: **binar.neo.pw** → IP **178.105.155.207**

Скрипт `deploy.sh` сам настраивает весь стек и **не конфликтует** с уже работающими на сервере проектами:

| Ресурс | Как обходим конфликт |
|---|---|
| Порт приложения | **3210** (3000 занят next-server, 8080 — другим приложением) |
| PostgreSQL (общий :5432) | отдельная роль и база `treebuilder`, чужие БД не трогаются |
| nginx (80/443) | добавляется **отдельный** server-блок только для `binar.neo.pw` |
| Приложение | слушает **127.0.0.1:3210** — наружу не торчит, только через nginx |

---

## Шаг 1. Настроить DNS (делается один раз, заранее)

В панели управления доменом `neo.pw` добавьте A-запись:

```
Тип: A
Имя: binar           (или binar.neo.pw — зависит от панели)
Значение: 178.105.155.207
TTL: 300
```

Проверить, что применилось:
```bash
dig +short binar.neo.pw      # должно вернуть 178.105.155.207
```

> Если на момент запуска скрипта DNS уже указывает на сервер — SSL-сертификат выпустится **автоматически**. Если нет — скрипт поднимет сайт по HTTP, а HTTPS включите потом одной командой (см. ниже).

---

## Шаг 2. Клонировать репозиторий и запустить деплой

На сервере под своим пользователем (НЕ root, но с правами sudo):

```bash
git clone <URL-вашего-репозитория> binar
cd binar
./deploy.sh
```

Скрипт сделает всё сам:
1. поставит Node.js 20 (если его нет);
2. создаст `.env` с автогенерированным паролем БД;
3. создаст роль и базу в PostgreSQL, применит миграции;
4. поднимет приложение как systemd-сервис `binar-tree` (автозапуск при перезагрузке);
5. настроит nginx для `binar.neo.pw`;
6. если DNS уже готов — выпустит SSL-сертификат Let's Encrypt.

---

## Шаг 3. Если SSL не выпустился автоматически

Когда DNS пропишется, выполните:
```bash
sudo certbot --nginx -d binar.neo.pw --agree-tos -m ok4ami@gmail.com --redirect
```
Сертификат продлевается автоматически (certbot ставит systemd-таймер).

---

## Управление после деплоя

```bash
sudo systemctl status binar-tree      # статус
sudo systemctl restart binar-tree     # перезапуск
sudo journalctl -u binar-tree -f      # логи в реальном времени
```

## Обновление кода (новая версия)

```bash
cd binar
git pull
./deploy.sh          # идемпотентно: переустановит зависимости, миграции, перезапустит сервис
```
Повторный запуск безопасен — пароль БД и существующие данные сохраняются.

## Что менять в конфиге

Все параметры — вверху `deploy.sh` (порт, имя БД, email, домен). Меняются там при необходимости.

---

## Firewall (если включён ufw)

Убедитесь, что открыты порты веба (порт приложения наружу НЕ нужен):
```bash
sudo ufw allow 'Nginx Full'    # 80 + 443
sudo ufw status
```
